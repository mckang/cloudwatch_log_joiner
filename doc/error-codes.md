# LogJoiner 에러 코드 표준


| 코드      | 분류      | 사용자 메시지                                     | 대표 원인                                         |
| ------- | ------- | ------------------------------------------- | --------------------------------------------- |
| `LJ001` | INPUT   | 입력값이 유효하지 않습니다. 인자/설정 파일 형식을 확인하세요.         | 잘못된 CLI 인자, YAML 필수 키 누락, 템플릿 포맷 오류           |
| `LJ101` | AWS     | CloudWatch 쿼리 실행 중 오류가 발생했습니다. 잠시 후 재시도하세요. | `start_query` 실패, 상태 `Failed/Timeout`, 재시도 소진 |
| `LJ201` | SQL     | 최종 SQL 실행에 실패했습니다. SQL 문법 및 참조 테이블을 확인하세요.  | DuckDB 실행 오류, `@timestamp` 누락/파싱 실패           |
| `LJ301` | FILE_IO | 파일 읽기/쓰기 중 오류가 발생했습니다. 경로와 권한을 확인하세요.       | 설정 파일 읽기 실패, 출력 디렉터리 초기화 실패, CSV 저장 실패        |


## 운영 원칙

- 콘솔에는 항상 `[코드] 사용자 메시지` 형태로 노출한다.
- 상세 원인은 `detail=` 필드에 포함해 트러블슈팅 가능한 수준으로 남긴다.
- 외부 연동 오류(AWS)는 재시도 이후에도 실패할 때만 최종 실패로 처리한다.
